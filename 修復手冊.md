## 問題一：代理錢包刪除不同步

### 🔍 症狀描述
- 手機上刪除代理錢包紀錄，電腦端沒有同步
- 或電腦端刪除，手機端沒有同步

### 🔍 根本原因
`deleteAgentWallet()` 函數中的同步邏輯有問題：
1. 刪除後沒有正確呼叫 `syncUpload()` 上傳到 Firebase
2. 或 listener 收到更新後沒有正確更新本地資料

### 🔧 診斷步驟

**步驟 1：檢查 Firebase 是否收到刪除指令**
```javascript
// 在瀏覽器偵錯工具 Console 執行
firebase.database().ref('macau_data/agentWallets').once('value').then(s => console.log(s.val()));
```

**步驟 2：檢查本地資料是否正確**
```javascript
// 在瀏覽器偵錯工具 Console 執行
console.log(JSON.parse(localStorage.getItem('macau_agent_wallets')));
```

**步驟 3：檢查 syncUpload 是否被呼叫**
在 `deleteAgentWallet()` 函數中加上：
```javascript
console.log('[Delete] 準備同步到 Firebase', agent, newArr);
```

### 🔧 解決方案

**方案 A：使用直接 set() 代替 transaction（推薦）**

修改 `deleteAgentWallet()` 函數：
```javascript
function deleteAgentWallet(agent, id) {
  var aw = getAgentWalletsLocalOnly();
  var arr = aw[agent] || [];
  var newArr = arr.filter(function(item) { return item.id !== id; });
  
  // 更新本地
  aw[agent] = newArr;
  saveAgentWalletsLocalOnly(aw);
  
  // 同步到 Firebase（使用直接 set 代替 transaction）
  if (newArr.length === 0) {
    db.ref('macau_data/agentWallets/' + agent).set(null, function(err) {
      if (err) console.error('[Sync ↑] 刪除失敗', err);
    });
  } else {
    var obj = {};
    for (var j = 0; j < newArr.length; j++) {
      var item = newArr[j];
      if (!item._fbKey) item._fbKey = _fbKey();
      obj[item._fbKey] = item;
    }
    db.ref('macau_data/agentWallets/' + agent).set(obj, function(err) {
      if (err) console.error('[Sync ↑] 同步失敗', err);
    });
  }
  
  renderAgentWallets();
}
```

**方案 B：確保 listener 正確更新 UI**

在 agentWallets 的 listener 中，確保刪除後會重新整理 UI：
```javascript
db.ref('macau_data/agentWallets').on('value', function(snapshot) {
  var d = snapshot.val();
  console.log('[Sync ↓] agentWallets 收到資料', d ? Object.keys(d).length + ' 個代理' : '空');
  
  // 修復：處理 null（全部刪除的情況）
  if (d === null) {
    console.log('[Sync ↓] Firebase 無資料，清空本地 agentWallets');
    agentWallets = {};
    saveAgentWalletsLocalOnly();
    try { doQuery(); } catch(e) {}
    return;
  }
  
  if (!d || typeof d !== 'object') return;
  
  // 轉換格式並更新本地
  var aw = {};
  var agents = Object.keys(d);
  for (var i = 0; i < agents.length; i++) {
    var agent = agents[i];
    var items = d[agent];
    if (!items || typeof items !== 'object') continue;
    aw[agent] = Object.values(items).map(function(item) {
      if (item._fbKey) _fbKey = item._fbKey; // 保留 _fbKey
      return item;
    });
  }
  
  agentWallets = aw;
  saveAgentWalletsLocalOnly();
  
  // 總是嘗試重新整理 UI
  try {
    var pq = document.getElementById("page-query");
    if (pq) { doQuery(); }
    var po = document.getElementById("page-overview");
    if (po && po.style.display === "block") { renderOverview(); }
  } catch(e) {}
});
```

### ✅ 驗證方法
1. 在手機上刪除一筆代理錢包紀錄
2. 在電腦上重新整理頁面（或等待自動同步）
3. 確認電腦上的該筆紀錄也消失
4. 反過來測試（電腦刪除 → 手機確認）

## 問題二：最後一筆記錄無法刪除

### 🔍 症狀描述
- 刪除代理的最後一筆錢包紀錄後，頁面重新整理又會回來
- 或刪除後立即停止同步功能

### 🔍 根本原因
有兩個可能的根本原因：

**原因 A：`_awArrToObj()` 跳過空陣列**
```javascript
function _awArrToObj(aw) {
  var result = {};
  var agents = Object.keys(aw);
  for (var i = 0; i < agents.length; i++) {
    var agent = agents[i];
    var arr = aw[agent] || [];
    if (!arr.length) continue; // ❌ 這行會導致空陣列被跳過
    result[agent] = {};
    for (var j = 0; j < arr.length; j++) {
      var item = arr[j];
      result[agent][item._fbKey] = item;
    }
  }
  return result;
}
```

**原因 B：Listener 收到 null 時直接 return**
```javascript
db.ref('macau_data/agentWallets').on('value', function(snapshot) {
  var d = snapshot.val();
  if (!d || typeof d !== 'object') return; // ❌ null 會在這裡被擋下
  // ...
});
```

### 🔧 診斷步驟

**步驟 1：檢查 Firebase 上的資料是否真的被刪除**
```javascript
// 在瀏覽器偵錯工具 Console 執行
firebase.database().ref('macau_data/agentWallets/代理名稱').once('value')
  .then(s => console.log('Firebase 資料:', s.val()));
```

**步驟 2：檢查 listener 是否有收到 null**
在 listener 中加上：
```javascript
console.log('[Sync ↓] agentWallets 收到資料', d ? Object.keys(d).length + ' 個代理' : '空');
```

### 🔧 解決方案

**修復 A：修改 `_awArrToObj()` 不跳過空陣列**
```javascript
function _awArrToObj(aw) {
  var result = {};
  var agents = Object.keys(aw);
  for (var i = 0; i < agents.length; i++) {
    var agent = agents[i];
    var arr = aw[agent] || [];
    // ✅ 修正：空陣列也要寫入（寫入空物件）
    result[agent] = {};
    for (var j = 0; j < arr.length; j++) {
      var item = arr[j];
      if (!item._fbKey) item._fbKey = _fbKey();
      result[agent][item._fbKey] = item;
    }
  }
  return result;
}
```

**修復 B：修改 listener 正確處理 null**
```javascript
db.ref('macau_data/agentWallets').on('value', function(snapshot) {
  var d = snapshot.val();
  console.log('[Sync ↓] agentWallets 收到資料', d ? Object.keys(d).length + ' 個代理' : '空');
  
  // ✅ 修正：d===null 表示 Firebase 上整個 agentWallets 已被清空
  if (d === null) {
    console.log('[Sync ↓] Firebase 無資料，清空本地 agentWallets');
    agentWallets = {};
    saveAgentWalletsLocalOnly();
    try { doQuery(); } catch(e) {}
    return;
  }
  
  if (!d || typeof d !== 'object') return;
  
  // ... 後續處理 ...
});
```

### ✅ 驗證方法
1. 新增一筆代理錢包紀錄
2. 刪除該筆紀錄（此時代理的錢包應該是空的）
3. 重新整理頁面
4. 確認該筆紀錄沒有再出現
5. 確認 Firebase 上的該代理資料是 `null`

## 問題三：版本更新後客戶端看不到新版本

### 🔍 症狀描述
- 部署新版本到 Railway，但電腦和手機都還是顯示舊版本
- 即使重新整理頁面也沒用

### 🔍 根本原因
**Service Worker (SW) 頑固快取**

瀏覽器註冊了 Service Worker (`sw.js`)，它會攔截網路請求並回傳快取版本的 `index.html`，導致客戶端永遠看不到新版本。

### 🔧 診斷步驟

**步驟 1：檢查是否有 Service Worker**
1. 按 `F12` 開啟偵錯工具
2. 點選 `Application` 標籤
3. 左側點選 `Service Workers`
4. 看是否有 `sw.js` 正在運行

**步驟 2：檢查 Network 請求**
1. 點選 `Network` 標籤
2. 重新整理頁面
3. 看 `index.html` 的 `Size` 欄位是否顯示 `(memory cache)` 或 `(disk cache)`

**步驟 3：檢查 Cache Storage**
1. 在 `Application` 標籤
2. 左側點選 `Cache Storage`
3. 看是否有快取 `index.html`

### 🔧 解決方案

**方案 A：手動清除 SW 和快取（立即見效）**

在客戶端的瀏覽器執行：
1. Chrome/Edge：
   - `F12` → `Application` → `Service Workers` → `Unregister`
   - `F12` → `Application` → `Cache Storage` → 右鍵刪除所有快取
   - `F12` → `Network` → 勾選 `Disable cache` → 重新整理

2. Safari (iOS)：
   - 設定 → Safari → 進階 → 網站資料 → 找到網址 → 刪除
   - 完全關閉 Safari 再重新打開

**方案 B：在程式碼中加入自動清除邏輯（推薦）**

在 `index.html` 的 `<head>` 最開頭加入：
```html
<script>
(function(){
  // 1. 強制解除所有已註冊的 Service Worker
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.getRegistrations().then(function(regs){
      regs.forEach(function(r){
        r.unregister();
        console.log('[SW] 已解除註冊:', r.scope);
      });
    });
  }
  // 2. 清除所有 Cache Storage
  if ('caches' in window) {
    caches.keys().then(function(keys){
      keys.forEach(function(key){
        caches.delete(key);
        console.log('[Cache] 已刪除:', key);
      });
    });
  }
})();
</script>
```

**方案 C：修改 `sw.js` 讓它自殺（推薦）**

將 `sw.js` 的內容改為：
```javascript
// 自殺式 Service Worker — 讓瀏覽器自動解除註冊
self.addEventListener('install', function() {
  self.skipWaiting();
  self.registration.unregister().then(function() {
    return self.clients.matchAll({ type: 'window' });
  }).then(function(clients) {
    clients.forEach(function(client) { client.navigate(client.url); });
  });
});
```

**方案 D：停止註冊新的 Service Worker**

在 `index.html` 中，找到並刪除這段程式碼：
```javascript
// ❌ 刪除這段
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('sw.js');
}
```

### ✅ 驗證方法
1. 部署新版本
2. 在客戶端開啟偵錯工具 (`F12`)
3. 看 `Console` 頁籤是否有 `[SW] 已解除註冊` 的訊息
4. 看登入頁面底部顯示的版本號是否為新版本

## 問題四：手機版顯示異常（需要一直滑動）

### 🔍 症狀描述
- 用手機開啟網頁時，需要一直左右滑動才能看到全部內容
- 網頁顯示比例異常（太小或太大）

### 🔍 根本原因
**缺少 `viewport` meta 標籤**

如果 `<head>` 中沒有 `viewport` meta 標籤，手機瀏覽器會用預設的 **980px 寬度**來渲染頁面，然後整個縮小顯示，導致使用者需要一直滑動才能看到全部內容。

### 🔧 診斷步驟

**步驟 1：檢查 HTML 原始碼**
在瀏覽器中 Right-click → `檢查` → `Elements` 標籤 → 看 `<head>` 內是否有：
```html
<meta name="viewport" content="...">
```

**步驟 2：用行動裝置模擬器檢查**
1. `F12` 開啟偵錯工具
2. 點選 `Toggle device toolbar` (Ctrl+Shift+M)
3. 選擇一個手機型號（例如 iPhone 12）
4. 看頁面是否正確縮放

### 🔧 解決方案

**在 `<head>` 中加入 viewport meta 標籤**

在 `index.html` 的 `<head>` 內，在第一個 `<meta>` 標籤後加入：
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
```

**參數說明**：
- `width=device-width`：用裝置的實際寬度來渲染（不是 980px）
- `initial-scale=1.0`：初始縮放比例為 1.0（不縮放）
- `maximum-scale=1.0`：最大縮放比例為 1.0（不允許放大）
- `user-scalable=no`：不允許使用者縮放（避免意外縮放）

### ✅ 驗證方法
1. 修改後部署新版本
2. 用手機開啟網頁
3. 確認不需要左右滑動就能看到全部內容
4. 確認文字和按鈕大小適中（不會太小或太大）

## 診斷工具與技巧

### 🔧 瀏覽器偵錯工具

**Chrome / Edge**
- 開啟偵錯工具：`F12` 或 `Ctrl+Shift+I`
- 清除快取：`Ctrl+Shift+R` 或 `Ctrl+F5`
- 停用快取：`F12` → `Network` → 勾選 `Disable cache`

**Safari (iOS)**
- 開啟偵錯工具：設定 → Safari → 進階 → 開啟「網頁檢閱器」
- 清除網站資料：設定 → Safari → 進階 → 網站資料 → 刪除全部

**Console 常用指令**
```javascript
// 檢查 Firebase 資料
firebase.database().ref('macau_data/agentWallets').once('value').then(s => console.log(s.val()));

// 檢查本地資料
console.log(JSON.parse(localStorage.getItem('macau_agent_wallets')));

// 手動觸發同步
syncUpload();

// 手動清除 Service Worker
navigator.serviceWorker.getRegistrations().then(r => r.forEach(r => r.unregister()));

// 手動清除 Cache
caches.keys().then(k => k.forEach(k => caches.delete(k)));
```

### 🔧 Firebase 偵錯

**在 Firebase Console 檢查資料**
1. 開啟 [Firebase Console](https://console.firebase.google.com/)
2. 選擇你的專案
3. 左側選單 → `Realtime Database`
4. 檢查 `macau_data/agentWallets` 的資料是否正確

**在程式碼中加入偵錯訊息**
```javascript
// 上傳時
console.log('[Sync ↑] 準備上傳', data);

// 下載時
console.log('[Sync ↓] 收到資料', snapshot.val());

// 錯誤時
db.ref(...).set(data, function(err) {
  if (err) console.error('[Sync] 錯誤', err);
});
```

## 預防措施

### ✅ 開發流程建議

**1. 每次修改後都要 commit 和 push**
```bash
# ❌ 錯誤：只執行 git add
git add index.html
# 沒有 commit 和 push，Railway 不會部署

# ✅ 正確：完整流程
git add index.html
git commit -m "v10.XX 修復 XXX 問題"
git push origin master
```

**2. 版本號要確實更新**
- 每次修改都在 `<title>` 和 `APP_VERSION` 更新版本號
- 版本號格式：`v10.21`, `v10.22`, ...
- 重大版本：`V1.0`, `V2.0`, ...

**3. 重要版本要備份**
```bash
# 在 render-deploy/ 目錄執行
cp index.html index_V1.0_GOLDEN.html
git add index_V1.0_GOLDEN.html
git commit -m "備份正式版 V1.0"
git push origin master
```

### ✅ Service Worker 管理建議

**1. 除非必要，否則不要使用 Service Worker**
- 這個系統需要聯網才能同步 Firebase，離線功能沒有實際價值
- Service Worker 會造成快取問題，讓客戶端看不到新版本

**2. 如果一定要用 Service Worker，要注意快取策略**
- 不要快取 `index.html`
- 每次部署都要更改 SW 的 cache name
- 在 `<head>` 中加入 SW 版本檢測邏輯

**3. 定期測試版本更新流程**
- 部署新版本後，用無痕模式開啟網頁
- 確認版本號是否正確
- 如果看不到新版本，立即檢查 SW 和快取

### ✅ 同步邏輯檢查清單

部署前，檢查以下項目：

- [ ] `deleteAgentWallet()` 有呼叫 `syncUpload()` 或使用 `set()`
- [ ] `saveAgentWallets()` 有同步到 Firebase
- [ ] Listener 有正確處理 `null`（全部刪除的情況）
- [ ] Listener 有重新整理 UI（`doQuery()` 或 `renderOverview()`）
- [ ] `_awArrToObj()` 不會跳過空陣列
- [ ] 沒有使用 `transaction()`（改用直接 `set()`）

## 📞 緊急處理 SOP

**如果客戶回報「刪除後又跑回來」**

1. **立即確認問題範圍**
   - 只有特定代理？還是所有代理？
   - 只有手機？還是電腦也一樣？

2. **檢查 Firebase 資料**
   ```javascript
   firebase.database().ref('macau_data/agentWallets/代理名稱').once('value')
     .then(s => console.log('Firebase:', s.val()));
   ```

3. **檢查本地資料**
   ```javascript
   console.log(JSON.parse(localStorage.getItem('macau_agent_wallets')));
   ```

4. **臨時解決方案**
   - 請客戶清除瀏覽器快取
   - 請客戶用無痕模式開啟
   - 如果問題持續，手動在 Firebase Console 刪除該筆資料

5. **長期解決方案**
   - 參考本文「問題一」和「問題二」的修復步驟
   - 修復後部署新版本
   - 通知所有使用者重新整理頁面
