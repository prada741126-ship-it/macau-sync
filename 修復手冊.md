## 問題一：代理錢包刪除不同步

### 🔍 症狀描述
- 手機上刪除代理錢包紀錄，電腦端沒有同步
- 或電腦端刪除，手機端沒有同步

### 🔍 根本原因
`deleteAgentWallet()` 函數中的同步邏輯有問題：
1. 刪除後沒有正確呼叫 `syncUpload()` 上傳到 Firebase
2. 或 listener 收到更新後沒有正確更新本地資料

### 🔧 診斷步驟

**步驟 1：檢查 Firebase 是否收到刪除指令**
```javascript
// 在瀏覽器偵錯工具 Console 執行
firebase.database().ref('macau_data/agentWallets').once('value').then(s => console.log(s.val()));
```

**步驟 2：檢查本地資料是否正確**
```javascript
// 在瀏覽器偵錯工具 Console 執行
console.log(JSON.parse(localStorage.getItem('macau_agent_wallets')));
```

**步驟 3：檢查 syncUpload 是否被呼叫**
在 `deleteAgentWallet()` 函數中加上：
```javascript
console.log('[Delete] 準備同步到 Firebase', agent, newArr);
```

### 🔧 解決方案

**方案 A：使用直接 set() 代替 transaction（推薦）**

修改 `deleteAgentWallet()` 函數：
```javascript
function deleteAgentWallet(agent, id) {
  var aw = getAgentWalletsLocalOnly();
  var arr = aw[agent] || [];
  var newArr = arr.filter(function(item) { return item.id !== id; });
  
  // 更新本地
  aw[agent] = newArr;
  saveAgentWalletsLocalOnly(aw);
  
  // 同步到 Firebase（使用直接 set 代替 transaction）
  if (newArr.length === 0) {
    db.ref('macau_data/agentWallets/' + agent).set(null, function(err) {
      if (err) console.error('[Sync ↑] 刪除失敗', err);
    });
  } else {
    var obj = {};
    for (var j = 0; j < newArr.length; j++) {
      var item = newArr[j];
      if (!item._fbKey) item._fbKey = _fbKey();
      obj[item._fbKey] = item;
    }
    db.ref('macau_data/agentWallets/' + agent).set(obj, function(err) {
      if (err) console.error('[Sync ↑] 同步失敗', err);
    });
  }
  
  renderAgentWallets();
}
```

**方案 B：確保 listener 正確更新 UI**

在 agentWallets 的 listener 中，確保刪除後會重新整理 UI：
```javascript
db.ref('macau_data/agentWallets').on('value', function(snapshot) {
  var d = snapshot.val();
  console.log('[Sync ↓] agentWallets 收到資料', d ? Object.keys(d).length + ' 個代理' : '空');
  
  // 修復：處理 null（全部刪除的情況）
  if (d === null) {
    console.log('[Sync ↓] Firebase 無資料，清空本地 agentWallets');
    agentWallets = {};
    saveAgentWalletsLocalOnly();
    try { doQuery(); } catch(e) {}
    return;
  }
  
  if (!d || typeof d !== 'object') return;
  
  // 轉換格式並更新本地
  var aw = {};
  var agents = Object.keys(d);
  for (var i = 0; i < agents.length; i++) {
    var agent = agents[i];
    var items = d[agent];
    if (!items || typeof items !== 'object') continue;
    aw[agent] = Object.values(items).map(function(item) {
      if (item._fbKey) _fbKey = item._fbKey; // 保留 _fbKey
      return item;
    });
  }
  
  agentWallets = aw;
  saveAgentWalletsLocalOnly();
  
  // 總是嘗試重新整理 UI
  try {
    var pq = document.getElementById("page-query");
    if (pq) { doQuery(); }
    var po = document.getElementById("page-overview");
    if (po && po.style.display === "block") { renderOverview(); }
  } catch(e) {}
});
```

### ✅ 驗證方法
1. 在手機上刪除一筆代理錢包紀錄
2. 在電腦上重新整理頁面（或等待自動同步）
3. 確認電腦上的該筆紀錄也消失
4. 反過來測試（電腦刪除 → 手機確認）

## 問題二：最後一筆記錄無法刪除

### 🔍 症狀描述
- 刪除代理的最後一筆錢包紀錄後，頁面重新整理又會回來
- 或刪除後立即停止同步功能

### 🔍 根本原因
有兩個可能的根本原因：

**原因 A：`_awArrToObj()` 跳過空陣列**
```javascript
function _awArrToObj(aw) {
  var result = {};
  var agents = Object.keys(aw);
  for (var i = 0; i < agents.length; i++) {
    var agent = agents[i];
    var arr = aw[agent] || [];
    if (!arr.length) continue; // ❌ 這行會導致空陣列被跳過
    result[agent] = {};
    for (var j = 0; j < arr.length; j++) {
      var item = arr[j];
      result[agent][item._fbKey] = item;
    }
  }
  return result;
}
```

**原因 B：Listener 收到 null 時直接 return**
```javascript
db.ref('macau_data/agentWallets').on('value', function(snapshot) {
  var d = snapshot.val();
  if (!d || typeof d !== 'object') return; // ❌ null 會在這裡被擋下
  // ...
});
```

### 🔧 診斷步驟

**步驟 1：檢查 Firebase 上的資料是否真的被刪除**
```javascript
// 在瀏覽器偵錯工具 Console 執行
firebase.database().ref('macau_data/agentWallets/代理名稱').once('value')
  .then(s => console.log('Firebase 資料:', s.val()));
```

**步驟 2：檢查 listener 是否有收到 null**
在 listener 中加上：
```javascript
console.log('[Sync ↓] agentWallets 收到資料', d ? Object.keys(d).length + ' 個代理' : '空');
```

### 🔧 解決方案

**修復 A：修改 `_awArrToObj()` 不跳過空陣列**
```javascript
function _awArrToObj(aw) {
  var result = {};
  var agents = Object.keys(aw);
  for (var i = 0; i < agents.length; i++) {
    var agent = agents[i];
    var arr = aw[agent] || [];
    // ✅ 修正：空陣列也要寫入（寫入空物件）
    result[agent] = {};
    for (var j = 0; j < arr.length; j++) {
      var item = arr[j];
      if (!item._fbKey) item._fbKey = _fbKey();
      result[agent][item._fbKey] = item;
    }
  }
  return result;
}
```

**修復 B：修改 listener 正確處理 null**
```javascript
db.ref('macau_data/agentWallets').on('value', function(snapshot) {
  var d = snapshot.val();
  console.log('[Sync ↓] agentWallets 收到資料', d ? Object.keys(d).length + ' 個代理' : '空');
  
  // ✅ 修正：d===null 表示 Firebase 上整個 agentWallets 已被清空
  if (d === null) {
    console.log('[Sync ↓] Firebase 無資料，清空本地 agentWallets');
    agentWallets = {};
    saveAgentWalletsLocalOnly();
    try { doQuery(); } catch(e) {}
    return;
  }
  
  if (!d || typeof d !== 'object') return;
  
  // ... 後續處理 ...
});
```

### ✅ 驗證方法
1. 新增一筆代理錢包紀錄
2. 刪除該筆紀錄（此時代理的錢包應該是空的）
3. 重新整理頁面
4. 確認該筆紀錄沒有再出現
5. 確認 Firebase 上的該代理資料是 `null`

## 問題三：版本更新後客戶端看不到新版本

### 🔍 症狀描述
- 部署新版本到 Railway，但電腦和手機都還是顯示舊版本
- 即使重新整理頁面也沒用

### 🔍 根本原因
**Service Worker (SW) 頑固快取**

瀏覽器註冊了 Service Worker (`sw.js`)，它會攔截網路請求並回傳快取版本的 `index.html`，導致客戶端永遠看不到新版本。

### 🔧 診斷步驟

**步驟 1：檢查是否有 Service Worker**
1. 按 `F12` 開啟偵錯工具
2. 點選 `Application` 標籤
3. 左側點選 `Service Workers`
4. 看是否有 `sw.js` 正在運行

**步驟 2：檢查 Network 請求**
1. 點選 `Network` 標籤
2. 重新整理頁面
3. 看 `index.html` 的 `Size` 欄位是否顯示 `(memory cache)` 或 `(disk cache)`

**步驟 3：檢查 Cache Storage**
1. 在 `Application` 標籤
2. 左側點選 `Cache Storage`
3. 看是否有快取 `index.html`

### 🔧 解決方案

**方案 A：手動清除 SW 和快取（立即見效）**

在客戶端的瀏覽器執行：
1. Chrome/Edge：
   - `F12` → `Application` → `Service Workers` → `Unregister`
   - `F12` → `Application` → `Cache Storage` → 右鍵刪除所有快取
   - `F12` → `Network` → 勾選 `Disable cache` → 重新整理

2. Safari (iOS)：
   - 設定 → Safari → 進階 → 網站資料 → 找到網址 → 刪除
   - 完全關閉 Safari 再重新打開

**方案 B：在程式碼中加入自動清除邏輯（推薦）**

在 `index.html` 的 `<head>` 最開頭加入：
```html
<script>
(function(){
  // 1. 強制解除所有已註冊的 Service Worker
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.getRegistrations().then(function(regs){
      regs.forEach(function(r){
        r.unregister();
        console.log('[SW] 已解除註冊:', r.scope);
      });
    });
  }
  // 2. 清除所有 Cache Storage
  if ('caches' in window) {
    caches.keys().then(function(keys){
      keys.forEach(function(key){
        caches.delete(key);
        console.log('[Cache] 已刪除:', key);
      });
    });
  }
})();
</script>
```

**方案 C：修改 `sw.js` 讓它自殺（推薦）**

將 `sw.js` 的內容改為：
```javascript
// 自殺式 Service Worker — 讓瀏覽器自動解除註冊
self.addEventListener('install', function() {
  self.skipWaiting();
  self.registration.unregister().then(function() {
    return self.clients.matchAll({ type: 'window' });
  }).then(function(clients) {
    clients.forEach(function(client) { client.navigate(client.url); });
  });
});
```

**方案 D：停止註冊新的 Service Worker**

在 `index.html` 中，找到並刪除這段程式碼：
```javascript
// ❌ 刪除這段
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('sw.js');
}
```

### ✅ 驗證方法
1. 部署新版本
2. 在客戶端開啟偵錯工具 (`F12`)
3. 看 `Console` 頁籤是否有 `[SW] 已解除註冊` 的訊息
4. 看登入頁面底部顯示的版本號是否為新版本

## 問題四：手機版顯示異常（需要一直滑動）

### 🔍 症狀描述
- 用手機開啟網頁時，需要一直左右滑動才能看到全部內容
- 網頁顯示比例異常（太小或太大）

### 🔍 根本原因
**缺少 `viewport` meta 標籤**

如果 `<head>` 中沒有 `viewport` meta 標籤，手機瀏覽器會用預設的 **980px 寬度**來渲染頁面，然後整個縮小顯示，導致使用者需要一直滑動才能看到全部內容。

### 🔧 診斷步驟

**步驟 1：檢查 HTML 原始碼**
在瀏覽器中 Right-click → `檢查` → `Elements` 標籤 → 看 `<head>` 內是否有：
```html
<meta name="viewport" content="...">
```

**步驟 2：用行動裝置模擬器檢查**
1. `F12` 開啟偵錯工具
2. 點選 `Toggle device toolbar` (Ctrl+Shift+M)
3. 選擇一個手機型號（例如 iPhone 12）
4. 看頁面是否正確縮放

### 🔧 解決方案

**在 `<head>` 中加入 viewport meta 標籤**

在 `index.html` 的 `<head>` 內，在第一個 `<meta>` 標籤後加入：
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
```

**參數說明**：
- `width=device-width`：用裝置的實際寬度來渲染（不是 980px）
- `initial-scale=1.0`：初始縮放比例為 1.0（不縮放）
- `maximum-scale=1.0`：最大縮放比例為 1.0（不允許放大）
- `user-scalable=no`：不允許使用者縮放（避免意外縮放）

### ✅ 驗證方法
1. 修改後部署新版本
2. 用手機開啟網頁
3. 確認不需要左右滑動就能看到全部內容
4. 確認文字和按鈕大小適中（不會太小或太大）

## 問題五：交易登錄後資料不同步

### 🔍 症狀描述
- 手機上新增交易紀錄，電腦端沒有同步
- 或電腦端新增，手機端沒有同步
- 交易資料在兩邊顯示不一致

### 🔍 根本原因

**原因 A：`saveData()` 沒有正確同步到 Firebase**

`saveData()` 函數中，新增或修改交易後，可能沒有呼叫 `syncUpload()` 或 `db.ref('macau_data/txs').set()`。

**原因 B：`macau_data` 的 listener 沒有正確更新本地資料**

Firebase 的 `on('value')` listener 可能沒有正確處理交易資料的更新。

**原因 C：交易 ID 或 `_fbKey` 不一致**

手機和電腦上的交易 ID 或 `_fbKey` 不匹配，導致 Firebase 上產生重複資料或資料遺失。

### 🔧 診斷步驟

**步驟 1：檢查 Firebase 是否有收到新交易**
```javascript
// 在瀏覽器偵錯工具 Console 執行
firebase.database().ref('macau_data/txs').once('value').then(s => {
  var d = s.val();
  console.log('Firebase 交易資料:', d ? Object.keys(d).length + ' 筆' : '空');
  if (d) console.log('第一筆資料:', Object.values(d)[0]);
});
```

**步驟 2：檢查本地資料是否正確**
```javascript
// 在瀏覽器偵錯工具 Console 執行
var txs = JSON.parse(localStorage.getItem('macau_data') || '[]');
console.log('本地交易資料:', txs.length + ' 筆');
if (txs.length > 0) console.log('第一筆資料:', txs[0]);
```

**步驟 3：檢查 `saveData()` 是否有呼叫同步**
在 `saveData()` 函數中加上：
```javascript
console.log('[Save] 準備同步到 Firebase', txs);
```

**步驟 4：檢查 listener 是否有收到更新**
在 `macau_data/txs` 的 listener 中加上：
```javascript
console.log('[Sync ↓] txs 收到資料', snapshot.val() ? Object.keys(snapshot.val()).length + ' 筆' : '空');
```

### 🔧 解決方案

**方案 A：確保 `saveData()` 有同步到 Firebase**

修改 `saveData()` 函數，在儲存本地資料後，立即同步到 Firebase：
```javascript
function saveData() {
  // ... 原有的儲存邏輯 ...
  
  // 同步到 Firebase（使用直接 set 代替 transaction）
  var obj = {};
  for (var i = 0; i < txs.length; i++) {
    var tx = txs[i];
    if (!tx._fbKey) tx._fbKey = _fbKey();
    obj[tx._fbKey] = tx;
  }
  
  db.ref('macau_data/txs').set(obj, function(err) {
    if (err) console.error('[Sync ↑] 同步失敗', err);
    else console.log('[Sync ↑] 同步成功', txs.length + ' 筆');
  });
}
```

**方案 B：確保 listener 正確更新本地資料**

在 `macau_data/txs` 的 listener 中，確保收到更新後會正確更新本地資料並刷新 UI：
```javascript
db.ref('macau_data/txs').on('value', function(snapshot) {
  var d = snapshot.val();
  console.log('[Sync ↓] txs 收到資料', d ? Object.keys(d).length + ' 筆' : '空');
  
  // 處理 null（全部刪除的情況）
  if (d === null) {
    console.log('[Sync ↓] Firebase 無資料，清空本地 txs');
    txs = [];
    saveLocalOnly();
    try { renderTable(); } catch(e) {}
    try { doQuery(); } catch(e) {}
    return;
  }
  
  if (!d || typeof d !== 'object') return;
  
  // 轉換格式並更新本地
  var newTxs = [];
  var keys = Object.keys(d);
  for (var i = 0; i < keys.length; i++) {
    var key = keys[i];
    var tx = d[key];
    tx._fbKey = key;
    newTxs.push(tx);
  }
  
  txs = newTxs;
  saveLocalOnly();
  
  // 總是嘗試重新整理 UI
  try {
    var pq = document.getElementById("page-query");
    if (pq && pq.style.display === "block") { doQuery(); }
    var po = document.getElementById("page-overview");
    if (po && po.style.display === "block") { renderOverview(); }
    var pa = document.getElementById("page-all");
    if (pa && pa.style.display === "block") { renderTable(); }
  } catch(e) {}
});
```

**方案 C：確保交易 ID 和 `_fbKey` 一致**

在 `saveData()` 和 listener 中，確保每筆交易都有正確的 `id` 和 `_fbKey`：
```javascript
// 在 saveData() 中
if (!tx.id) tx.id = Date.now();  // 如果沒有 id，用時間戳記作為 id
if (!tx._fbKey) tx._fbKey = _fbKey();  // 如果沒有 _fbKey，產生一個新的

// 在 listener 中
tx._fbKey = key;  // 確保 _fbKey 與 Firebase 的 key 一致
```

### ✅ 驗證方法
1. 在手機上新增一筆交易紀錄
2. 在電腦上重新整理頁面（或等待自動同步）
3. 確認電腦上有出現該筆紀錄
4. 反過來測試（電腦新增 → 手機確認）
5. 檢查 Firebase Console，確認資料正確

## 問題六：交易刪除後又跑回來

### 🔍 症狀描述
- 刪除交易紀錄後，頁面重新整理又會跑回來
- 或刪除後立即停止同步功能

### 🔍 根本原因

**原因 A：`deleteTx()` 沒有正確同步到 Firebase**

`deleteTx()` 函數中，刪除本地資料後，可能沒有呼叫 `syncUpload()` 或 `db.ref('macau_data/txs/' + _fbKey).set(null)`。

**原因 B：Listener 收到更新後沒有正確更新本地資料**

Firebase 的 `on('value')` listener 可能沒有正確處理刪除後的更新。

**原因 C：最後一筆刪除後，Firebase 回傳 `null`，但 listener 沒有正確處理**

類似「問題二」的情況，`snapshot.val()` 回傳 `null` 時，程式碼直接 `return`，沒有清空本地資料。

### 🔧 診斷步驟

**步驟 1：檢查 Firebase 上的資料是否真的被刪除**
```javascript
// 在瀏覽器偵錯工具 Console 執行
firebase.database().ref('macau_data/txs/' + _fbKey).once('value')
  .then(s => console.log('Firebase 資料:', s.val()));
```

**步驟 2：檢查 listener 是否有收到 `null`**
在 listener 中加上：
```javascript
console.log('[Sync ↓] txs 收到資料', d ? Object.keys(d).length + ' 筆' : '空');
```

**步驟 3：檢查 `deleteTx()` 是否有呼叫同步**
在 `deleteTx()` 函數中加上：
```javascript
console.log('[Delete] 準備同步到 Firebase', id, _fbKey);
```

### 🔧 解決方案

**方案 A：確保 `deleteTx()` 有同步到 Firebase**

修改 `deleteTx()` 函數，在刪除本地資料後，立即同步到 Firebase：
```javascript
function deleteTx(id) {
  var idx = -1;
  for (var i = 0; i < txs.length; i++) {
    if (txs[i].id === id) { idx = i; break; }
  }
  if (idx < 0) return;
  
  var tx = txs[idx];
  var _fbKey = tx._fbKey;
  
  // 刪除本地資料
  txs.splice(idx, 1);
  saveLocalOnly();
  
  // 同步到 Firebase（使用直接 set 代替 transaction）
  if (_fbKey) {
    db.ref('macau_data/txs/' + _fbKey).set(null, function(err) {
      if (err) console.error('[Sync ↑] 刪除失敗', err);
      else console.log('[Sync ↑] 刪除成功', id);
    });
  } else {
    // 如果沒有 _fbKey，就整個重新上傳
    syncUpload();
  }
  
  renderTable();
}
```

**方案 B：確保 listener 正確處理 `null`**

在 `macau_data/txs` 的 listener 中，加入 `null` 處理邏輯（參考「問題二」的修復方案）。

### ✅ 驗證方法
1. 新增一筆交易紀錄
2. 刪除該筆紀錄
3. 重新整理頁面
4. 確認該筆紀錄沒有再出現
5. 確認 Firebase 上的該筆資料是 `null`

## 問題七：交易計算錯誤（佣金/碼糧/公基金）

### 🔍 症狀描述
- 佣金（comm）計算錯誤
- 碼糧（bonus）計算錯誤
- 公基金（fund）計算錯誤
- 未提領（undrawn）計算錯誤

### 🔍 根本原因

**原因 A：`calcComm()` 或 `calcFund()` 的公式錯誤**

計算公式可能寫錯，或沒有正確處理邊界情況（例如：洗碼量為 0、碼佣率為 0 等）。

**原因 B：表單送出時沒有正確計算**

`saveData()` 或表單送出邏輯中，可能沒有正確呼叫計算函數，或計算結果沒有正確寫入交易物件。

**原因 C：顯示時沒有正確格式化**

UI 顯示時，可能沒有正確格式化數字（例如：沒有千分位、沒有小數點等）。

### 🔧 診斷步驟

**步驟 1：檢查計算公式是否正確**
```javascript
// 在瀏覽器偵錯工具 Console 執行
console.log('calcComm(10, 1.25) =', calcComm(10, 1.25));  // 應該是 Math.ceil(10 * 10000 * 0.0125) = 1250
console.log('calcFund(1250, 1000) =', calcFund(1250, 1000));  // 應該是 1250 - 1000 = 250
```

**步驟 2：檢查表單送出時是否有正確計算**
在 `saveData()` 或表單送出邏輯中加上：
```javascript
console.log('[Save] 計算前', vol, rate);
var comm = calcComm(vol, rate);
var bonus = toNum(document.getElementById('bonus').value) || 0;
var fund = calcFund(comm, bonus);
console.log('[Save] 計算後', comm, bonus, fund);
```

**步驟 3：檢查 Firebase 上的資料是否正確**
```javascript
// 在瀏覽器偵錯工具 Console 執行
firebase.database().ref('macau_data/txs').once('value').then(s => {
  var d = s.val();
  var keys = Object.keys(d);
  var tx = d[keys[0]];
  console.log('第一筆資料:', tx);
  console.log('佣金:', tx.comm, '碼糧:', tx.bonus, '公基金:', tx.fund);
});
```

### 🔧 解決方案

**方案 A：修正計算公式**

確認 `calcComm()` 和 `calcFund()` 的公式正確：
```javascript
function calcComm(vol, rate) {
  // 佣金 = 洗碼量 × 10000 × 碼佣率% → Math.ceil()
  return Math.ceil(vol * 10000 * rate / 100);
}

function calcFund(comm, bonus) {
  // 公基金 = 佣金 - 碼糧
  return comm - bonus;
}
```

**方案 B：確保表單送出時有正確計算**

在 `saveData()` 或表單送出邏輯中，確保計算結果有正確寫入交易物件：
```javascript
function saveData() {
  // ... 讀取表單資料 ...
  
  var vol = toNum(document.getElementById('volume').value);
  var rate = toNum(document.getElementById('rate').value);
  var bonus = toNum(document.getElementById('bonus').value) || 0;
  
  var comm = calcComm(vol, rate);
  var fund = calcFund(comm, bonus);
  var undrawn = Math.max(0, comm - bonus - drawn);
  
  var tx = {
    // ... 其他欄位 ...
    volume: vol,
    rate: rate,
    comm: comm,
    bonus: bonus,
    fund: fund,
    undrawn: undrawn,
    // ... 其他欄位 ...
  };
  
  // ... 儲存邏輯 ...
}
```

**方案 C：確保顯示時有正確格式化**

在 UI 顯示時，使用 `fmt()` 函數來格式化數字：
```javascript
// 在 renderTable() 或 doQuery() 中
html += '<td>' + fmt(tx.comm) + '</td>';
html += '<td>' + fmt(tx.bonus) + '</td>';
html += '<td>' + fmt(tx.fund) + '</td>';
```

### ✅ 驗證方法
1. 新增一筆交易紀錄，輸入洗碼量 10、碼佣率 1.25
2. 確認佣金計算是 1,250（10 × 10000 × 1.25% = 1250）
3. 確認碼糧、公基金、未提領的計算也正確
4. 檢查 Firebase Console，確認資料正確

## 問題八：交易資料在電腦手機顯示不一致

### 🔍 症狀描述
- 電腦和手機上看到的交易資料不一致
- 同一筆交易，在電腦和手機上顯示的日期、佣金、碼糧等不同
- 電腦和手機的總計金額不同

### 🔍 根本原因

**原因 A：日期或時間處理不一致**

電腦和手機的時區設定不同，導致 `getDow()` 或日期處理函數回傳不同的結果。

**原因 B：數字格式化不一致**

電腦和手機的數字格式化方式不同（例如：千分位分隔符號、小數點等）。

**原因 C：Firebase 同步不完整**

Firebase 同步過程中，可能有一部分資料遺失或重複。

### 🔧 診斷步驟

**步驟 1：檢查電腦和手機的本地資料是否一致**
```javascript
// 在電腦的瀏覽器偵錯工具 Console 執行
var txsPC = JSON.parse(localStorage.getItem('macau_data') || '[]');
console.log('電腦本地資料:', txsPC.length + ' 筆');

// 在手機的瀏覽器偵錯工具 Console 執行
var txsMobile = JSON.parse(localStorage.getItem('macau_data') || '[]');
console.log('手機本地資料:', txsMobile.length + ' 筆');
```

**步驟 2：檢查 Firebase 上的資料是否正確**
```javascript
// 在電腦的瀏覽器偵錯工具 Console 執行
firebase.database().ref('macau_data/txs').once('value').then(s => {
  var d = s.val();
  console.log('Firebase 資料:', d ? Object.keys(d).length + ' 筆' : '空');
});
```

**步驟 3：檢查日期處理是否一致**
```javascript
// 在電腦和手機的瀏覽器偵錯工具 Console 都執行
console.log('現在時間:', new Date().toString());
console.log('getDow(今天)', getDow(nowStr()));
```

### 🔧 解決方案

**方案 A：確保日期處理一致**

使用 `nowStr()` 函數來取得 GMT+8 的日期字串，確保電腦和手機的日期一致：
```javascript
function nowStr() {
  var now = new Date();
  var offset = 8 * 60;  // GMT+8
  var local = new Date(now.getTime() + (offset - now.getTimezoneOffset()) * 60000);
  return local.toISOString().split('T')[0];
}
```

**方案 B：確保數字格式化一致**

使用 `fmt()` 函數來格式化數字，確保電腦和手機的顯示一致：
```javascript
function fmt(n) {
  if (n === null || n === undefined || isNaN(n)) return '0';
  return Number(n).toLocaleString('zh-TW');
}
```

**方案 C：確保 Firebase 同步完整**

參考「問題五」的修復方案，確保 `saveData()` 和 listener 都正確處理 Firebase 同步。

### ✅ 驗證方法
1. 在電腦和手機上都開啟網頁
2. 確認兩邊顯示的交易資料一致
3. 確認日期、佣金、碼糧、公基金等欄位都一致
4. 確認總計金額一致

## 預防措施

### ✅ 開發流程建議

**1. 每次修改後都要 commit 和 push**
```bash
# ❌ 錯誤：只執行 git add
git add index.html
# 沒有 commit 和 push，Railway 不會部署

# ✅ 正確：完整流程
git add index.html
git commit -m "v10.XX 修復 XXX 問題"
git push origin master
```

**2. 版本號要確實更新**
- 每次修改都在 `<title>` 和 `APP_VERSION` 更新版本號
- 版本號格式：`v10.21`, `v10.22`, ...
- 重大版本：`V1.0`, `V2.0`, ...

**3. 重要版本要備份**
```bash
# 在 render-deploy/ 目錄執行
cp index.html index_V1.0_GOLDEN.html
git add index_V1.0_GOLDEN.html
git commit -m "備份正式版 V1.0"
git push origin master
```

### ✅ Service Worker 管理建議

**1. 除非必要，否則不要使用 Service Worker**
- 這個系統需要聯網才能同步 Firebase，離線功能沒有實際價值
- Service Worker 會造成快取問題，讓客戶端看不到新版本

**2. 如果一定要用 Service Worker，要注意快取策略**
- 不要快取 `index.html`
- 每次部署都要更改 SW 的 cache name
- 在 `<head>` 中加入 SW 版本檢測邏輯

**3. 定期測試版本更新流程**
- 部署新版本後，用無痕模式開啟網頁
- 確認版本號是否正確
- 如果看不到新版本，立即檢查 SW 和快取

### ✅ 同步邏輯檢查清單

部署前，檢查以下項目：

- [ ] `deleteAgentWallet()` 有呼叫 `syncUpload()` 或使用 `set()`
- [ ] `saveAgentWallets()` 有同步到 Firebase
- [ ] Listener 有正確處理 `null`（全部刪除的情況）
- [ ] Listener 有重新整理 UI（`doQuery()` 或 `renderOverview()`）
- [ ] `_awArrToObj()` 不會跳過空陣列
- [ ] 沒有使用 `transaction()`（改用直接 `set()`）
