## 系統架構

### 技術棧
- **前端**：單一 HTML 檔案（內嵌 CSS + JavaScript）
- **後端同步**：Firebase Realtime Database（v7.0+）
- **舊版後端**：Railway + server.js（v6.x，已廢棄）
- **加密**：AES 加密（交易資料、公基金、代理錢包）
- **圖表**：Chart.js（趨勢圖、代理排行圖）

## 核心計算公式

### 1. 佣金計算
```javascript
function calcComm(vol, rate) {
  // 洗碼量(vol) 單位：萬
  // 碼佣率(rate) 單位：%
  // 公式：佣金 = 洗碼量 × 10000 × 碼佣率% → 無條件進位(Math.ceil)
  return Math.ceil(vol * 10000 * rate / 100);
}
```

### 2. 公基金計算
```javascript
function calcFund(comm, bonus) {
  // 公基金 = 佣金 - 碼糧
  return comm - bonus;
}
```

### 3. 未提領計算
```javascript
// 未提領 = max(0, 碼糧 - 已提領)
undrawn = Math.max(0, bonus - drawn);
```

### 4. 洗碼量內部儲存
```
洗碼量內部儲存 = 萬為單位（10 = 10萬）
```

```javascript
  id: Number,       // 自增 ID

  comm: Number,     // 佣金

  drawn: Number,    // 已提領
  undrawn: Number,  // 未提領（自動計算）
```

```javascript
  appId: "1:1020087145693:web:92b3a8c9f31d33c1fc9e6e"

```

### 同步流程

#### 上傳（syncUpload）
1. 檢查 `db` 是否存在（Firebase 是否初始化）
2. 設定 `localEditLockUntil = Date.now() + 3000`（鎖定 3 秒）
3. 組裝 payload（包含 txs, agentList, agentWallets, fundWithdrawals, workingMonth, rmBookings, archives, timestamp）
4. 使用 `db.ref('macau_data').set(payload)` 上傳到 Firebase
5. 上傳成功後，3 秒內忽略所有 Firebase 更新（防止競態條件）

#### 下載（initSync）
1. 使用 `db.ref('macau_data').on('value', callback)` 監聽 Firebase 資料變化
2. 收到資料時，檢查時間戳：
   - 如果 `incomingTs < lastSyncTimestamp`，跳過（舊資料）
   - 如果 `Date.now() < localEditLockUntil`，跳過（本地編輯鎖定中）
3. 更新本地變數（txs, agentList, agentWallets, fundWithdrawals, workingMonth, rmBookings）
4. 儲存到 localStorage（加密備份）
5. 重新渲染所有頁面（renderAll, renderOverview, renderSummary, RM.render, etc.）
6. 更新 `lastSyncTimestamp = incomingTs`

### 防止無限循環機制
1. **isSyncingFromFirebase 標誌**（v7.1）：
   - `initSync()` 開始時設定 `isSyncingFromFirebase = true`
   - `syncUpload()` 開頭檢查，如果 `isSyncingFromFirebase === true`，跳過上傳
   - `initSync()` 結束時（finally 區塊）設定 `isSyncingFromFirebase = false`

2. **localEditLockUntil 機制**（v7.3+）：
   - `syncUpload()` 上傳後設定 `localEditLockUntil = Date.now() + 3000`
   - `initSync()` 收到 Firebase 更新時，檢查 `Date.now() < localEditLockUntil`
   - 如果為 true，跳過 Firebase 更新（2 秒內）
   - **這個機制同時防止了無限循環**（上傳後 2 秒內 Firebase 回調會被忽略）

## 安全性功能

### 1. AES 加密
- **加密對象**：交易資料（txs）、公基金紀錄（fundWithdrawals）、代理錢包（agentWallets）
- **未加密對象**：代理名單（agentList）、macau_config、macau_archives、rm_bookings
- **加密函數**：`encryptData(obj)`、`decryptData(str)`
- **金鑰**：內嵌在 `index.html` 中（請勿外流）

### 2. 密碼保護
- **登入密碼**：`macau888`（Base64 編碼儲存）
- **失敗 3 次鎖定 30 秒**，之後遞增鎖定時間
- **會話超時**：30 分鐘無操作自動登出
- **遠端訪問檢測**：`isRemoteAccess()` 檢測是否為遠端訪問，如果是則顯示密碼覆蓋層

### 3. Firebase 安全規則（建議設定）
```javascript
{
  "rules": {
    ".read": true,   // 公開讀取（測試模式）
    ".write": true   // 公開寫入（測試模式）
  }
}
```
**注意**：目前是測試模式（公開訪問）。建議改用 Firebase Auth 或設定更嚴格的安全規則。

### 4. 本地編輯鎖定（v7.3+）
- **目的**：防止 Firebase 舊資料覆蓋剛刪除的資料（競態條件）
- **機制**：`syncUpload()` 上傳後設定 `localEditLockUntil = Date.now() + 3000`
- **效果**：3 秒內忽略所有 Firebase 更新

## 常見問題（FAQ）

### Q1：刪除交易後，資料又出現？
**A**：這是競態條件（Race Condition）。已於 v7.3 修復，添加 `localEditLockUntil` 機制（本地編輯後鎖定 3 秒）。

### Q2：頁面卡死、無法做任何動作？
**A**：這是 Firebase 同步無限循環。已於 v7.1 修復，添加 `isSyncingFromFirebase` 標誌防止無限循環。

### Q3：Chart.js 圖表顯示「Canvas is already in use」錯誤？
**A**：已於 v7.4 修復，使用 `Chart.getChart(canvas)` 檢查並銷毀舊圖表實例。

### Q4：如何強制刷新頁面（清除快取）？
**A**：
- Windows：`Ctrl + F5`
- Mac：`Cmd + Shift + R`

### Q5：如何新增代理？
**A**：總覽頁面 → 點擊「代理管理」→ 輸入代理名稱 → 點擊「新增」。

### Q6：如何執行月末結算？
**A**：總覽頁面 → 點擊「月末結算」→ 確認 → 系統會封存當月資料到 `macau_archives`。
